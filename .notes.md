
## Lovable Deploy Issue (2026-04-09)

**Problema:** Após push no GitHub, o site em produção (adbrief.pro) não atualizava — mesmo com hard refresh, aba anônima, ou query string. Preview do Lovable mostrava correto, mas o domínio de produção não.

**Solução:** Enviar um commit vazio (`git commit --allow-empty`) e fazer push. Depois republicar manualmente no Lovable. Isso força o Lovable a reconhecer a mudança e triggerar um novo build/deploy.

**Comando:**
```bash
git commit --allow-empty -m "Trigger rebuild" && git push origin main
```

**Nota:** Sempre que o Lovable não pegar as mudanças após push, usar esse workaround antes de perder tempo debugando cache.
